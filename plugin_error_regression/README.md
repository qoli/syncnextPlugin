# plugin_error_regression

這是一個專用於 Syncnext UI 回歸測試的插件。

它會在封面列表階段保留這句錯誤：

`封面列表為空：站點頁面結構可能變更，或目前返回了非預期內容。`

同時它會返回多個可點擊的封面入口，用來測試不同階段的錯誤頁。

目前內建場景：

1. `首頁錯誤殘留檢查`
   首頁保留 coverList 錯誤，但詳情頁仍返回 1 集，用來驗證首頁插件錯誤不會串場到結果頁或其它來源。
2. `劇集 emptyView`
   劇集階段主動回報 `emptyView`，用來測試劇集錯誤頁。
3. `劇集空陣列`
   劇集階段直接返回空陣列，不附帶 `emptyView`，用來測試通用空白頁。
4. `播放器失效地址`
   播放器階段返回失效播放地址，用來測試播放器錯誤頁。
5. `播放器 emptyView`
   播放器階段主動回報 `emptyView`，不返回播放地址。
6. `候選播放全部失敗`
   播放器階段返回多個候選地址，但全部失效。
7. `播放器空字串`
   播放器階段直接返回空字串，用來測試未取得播放地址。

用途：

1. 在 Syncnext 中打開這個插件。
2. 確認首頁能看到多個封面項目，且首頁仍保留 coverList 錯誤。
3. 依序點擊不同封面入口，驗證劇集頁與播放器頁的錯誤報告是否正確區分。
4. 不重啟 App，直接再打開原生頻道或 VOD 頻道。
5. 驗證插件錯誤不會串場顯示到原生或 VOD 的結果頁 / 播放器頁。

建議把這個插件 config 透過本地 HTTP 服務或 GitHub Raw 地址接入 Syncnext。
